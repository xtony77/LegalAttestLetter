## Context

全新專案，目前無任何程式碼。本階段僅建立 UI Mockup，確認表單互動與視覺風格後再進入 PDF 產生功能的實作。目標使用者包含不熟悉技術的一般民眾（含長輩），因此 UX 必須直覺簡潔。

## Goals / Non-Goals

**Goals:**
- 建立可運行的 React SPA，展示完整的表單 UI 與互動流程
- 響應式設計，桌面與手機皆可操作
- 確立視覺設計語言（正義女神意象：深靛藍 + 金色 + 淺灰白）
- 表單驗證（必填欄位提示）
- 動態新增/刪除寄件人、收件人、副本收件人

**Non-Goals:**
- PDF 產生功能（後續 change）
- 字型載入與嵌入（後續 change）
- 後端 API、資料庫、使用者帳號
- 存證信函內文預覽畫面

## Decisions

### 1. 框架選擇：Vite + React 19 + TypeScript

選 Vite 而非 Next.js。本專案是純前端 SPA，不需要 SSR/SSG。Vite 建置快、設定少，產出靜態檔案直接部署 Cloudflare Pages。作為面試作品也展示了「選對工具」的判斷力。

### 2. CSS 框架：Tailwind CSS v4

使用者指定。v4 是最新主流版本，utility-first 適合快速迭代 UI，內建響應式斷點。

### 3. 表單狀態管理：React 內建 state

表單結構簡單（三組動態陣列 + 一個文字框），不需要引入 Redux/Zustand 等狀態管理庫。使用 `useState` + 簡單的陣列操作即可。

### 4. UI 佈局策略

**桌面版（≥768px）：**
- 單欄居中卡片式佈局，最大寬度 720px
- 寄件人與收件人並排兩欄
- 副本收件人獨立一列
- 內文輸入框全寬

**手機版（<768px）：**
- 全寬單欄堆疊
- 寄件人、收件人、副本收件人依序垂直排列
- 內文輸入框全寬

### 5. 色彩系統

| Token | 色碼 | 用途 |
|-------|------|------|
| primary | #1B365D | 深靛藍，主色調（標題、按鈕、邊框） |
| accent | #C4A265 | 金色，強調元素（圖示、hover 效果） |
| background | #F5F5F0 | 淺灰白，頁面背景 |
| surface | #FFFFFF | 白色，卡片/輸入框背景 |
| text | #2D2D2D | 深灰，主要文字 |
| text-muted | #6B7280 | 次要文字、placeholder |
| success | #2E7D5B | 成功/確認 |
| error | #DC2626 | 錯誤/驗證提示 |

### 6. 動態欄位的新增/刪除互動

- 每個角色（寄件人/收件人/副本收件人）預設顯示 1 組欄位
- 「+ 新增」按鈕在該區塊底部
- 每組欄位右上角有刪除按鈕（當數量 > 1 時顯示，寄件人和收件人至少保留 1 組）
- 副本收件人可以全部刪除（選填）

## Risks / Trade-offs

- **[Risk] Tailwind v4 較新，部分生態系套件可能尚未完全相容** → 本專案不依賴第三方 UI 元件庫，直接用 Tailwind utility class 手刻元件，不受影響。
- **[Risk] 此階段「產生 PDF」按鈕無實際功能** → 按鈕加上 disabled 或 tooltip 提示「功能開發中」，避免使用者困惑。
